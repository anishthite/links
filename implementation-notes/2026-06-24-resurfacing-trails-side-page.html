Algorithm

  1. Prefer link-backed notes with at least one display title or source URL; fall back to ordinary notes when sparse.
  2. Score resurfacing with score = ageDays*1.2 + contentSignal + tagSignal + hostSignal - touchedPenalty, where ageDays uses the oldest valid source/created/updated timestamp.
  3. Build trails by picking seed notes with enough term/tag overlap, then append nearest related notes until each trail has three steps and skip reused seeds.
  4. Refresh by refetching notes and rotating within the top 18 ranked candidates by offset = variant*3 % pool.length.
  5. Explain each card with visible factors: age, host, tags, extract length, and shared terms.

UI layout

  1. Briefing header: page title, corpus count, and a board-return link.
  2. Refresh control: top-bar action advances the ranked-pool variant and refetches notes.
  3. Resurface rail: three ranked links with rationale, tags, and source actions.
  4. Trail section: ordered paths with numbered steps, connector rules, and shared-topic labels.
  5. Debug strip: compact signal chips expose why the deterministic picks happened.

Decisions

IDChoice · Rationale
D-001Use existing getNotes(). The experiment needs no write path or schema.
D-002Render as a side page. The board keeps its default first-screen behavior.
D-003Use deterministic local scoring. The first pass stays inspectable and testable without model latency.
D-004Fall back from source notes to ordinary notes. vite-only dev has no link-backed sample data.
D-005Refresh rotates the high-signal pool. Refetch-only would return the same deterministic list.
D-006Deploy the Worker without container rollout. Docker was unavailable locally and this UI/API change does not require updating links-sandbox.

Tradeoffs

IDPicked over Rejected · Reason
T-001/resurface route over a new header mode. Side-page routing avoids disturbing saved board view state.
T-002Lexical/tag overlap over embeddings. Current client has full note extracts but no runtime vector index.
T-003Variant rotation over random shuffle. Refresh stays biased toward high-signal candidates.
