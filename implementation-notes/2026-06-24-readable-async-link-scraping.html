API contract

Request

POST /api/notes
{
  text?: string,
  tags?: string[],
  sourceUrl?: string
}

Response — link-backed note

{
  note: {
    uuid: string,
    sourceUrl: string,
    sourceStatus: "pending" | "ready" | "failed",
    sourceExtractor: string | null,
    sourceContentLength: number | null,
    sourceContentTruncated: 0 | 1 | null
  }
}

Algorithm

  1. Create a pending row with normalized URL, user text, tags, and host fallback metadata.
  2. Run extraction in ExecutionContext.waitUntil; tests can call the scraper directly.
  3. Try YouTube transcript, then local Readability, then optional external scraper, then old body-strip fallback.
  4. Store note preview text plus telemetry on notes; store full-ish text windows in note_source_chunks.
  5. Mark source_content_truncated = 1 when the extract exceeds the stored preview cap.

Decisions

IDChoice · Rationale
D-001Keep local Readability as default. Avoid paid external dependency for ordinary pages.
D-002Make external scraper opt-in through env. Secrets and per-request costs stay off the default path.
D-003Keep a capped preview in notes.source_content_text. Existing UI and chat paths keep working while chunks carry longer extracts.
D-004Use waitUntil over Cloudflare Queues. The app needs one-shot background work, not a durable queue yet.

Tradeoffs

IDPicked over Rejected · Reason
T-001note_source_chunks over raising source_content_text indefinitely. Avoids bloating every list response.
T-002Readability + fallback over browser rendering. Captures most articles without container latency.
T-003Firecrawl/Jina adapter over SDK dependency. Worker code only needs one HTTP call.

Assumptions

  • A-001 · Most saved links are public article-like pages. If wrong: enable LINK_SCRAPER_PROVIDER for browser-grade extraction.
  • A-002 · A stale pending note is acceptable when background extraction fails. If wrong: add retry cron over source_status = 'pending'.

Reversibility

R-001 · medium. Revert route/extractor code and ignore additive schema columns/table; old rows remain readable.

Followups

  • L-001 · Backfill chunks for existing long extracts after the migration lands in production.
  • L-002 · Promote external scraper fallback after measuring failure domains in saved links.

Definition of done

  • New link notes return immediately with source_status = 'pending'.
  • Background extraction updates source metadata and writes chunks.
  • Refresh rebuilds source metadata and chunks synchronously.
  • npm test -- tests/link-source.test.ts and npm run typecheck pass.
