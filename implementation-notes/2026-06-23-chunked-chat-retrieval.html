Algorithm

  1. Tokenize the query plus recent chat with existing stopword filtering.
  2. Score note metadata with the existing title/tag/text weights.
  3. Split long sourceContentText into ~1,200-character windows with ~200-character overlap.
  4. Score each chunk with the same lexical scorer plus metadata boost.
  5. Keep the best chunk per note; return top notes with sourceContentText replaced by the winning chunk for prompting.

Decisions

D-001Chunk at query time. Avoids schema/index backfill before a local golden eval exists.
D-002Keep one best chunk per note. Existing UI opens notes by uuid; duplicate source cards would add noise.
D-003Use lexical scoring first. Current corpus is small enough to scan and the next useful step is evidence focus, not vector infra.
