API contract

Request

POST /api/links
{ url: string, tags?: string[], note?: string }

PATCH /api/links/:id
{ tags?: string[], note?: string, title?: string }

POST /api/links/:id/refresh
{}

POST /api/chat
{ message: string }

Response — LinkWire

{
  id: string,
  url: string,
  normalizedUrl: string,
  title: string,
  description: string,
  siteName: string,
  tags: string[],
  note: string,
  contentText: string,
  savedAt: number,
  fetchedAt: number,
  status: "ready" | "failed"
}

Algorithm

  1. Normalize URL; reject invalid or duplicate normalized_url.
  2. Fetch page server-side; extract <title>, meta description, site, and body text with script/style stripping.
  3. Store link row; mirror searchable fields into links_fts.
  4. Search via FTS MATCH; fall back to recent rows when query is empty.
  5. Answer questions by retrieving top matches, then synthesize with local snippets or model text when creds exist.

UI layout

  1. Header: app title, short explainer, search box.
  2. Add form: URL input, tags input, note textarea, submit.
  3. Results list: card per link with title, site, tags, note, excerpt, actions.
  4. Chat panel: ask a question; show answer and cited links.

Decisions

IDChoice · Rationale
D-001Keep tags as JSON in links.tags. Faster MVP; no join table yet.
D-002Store extracted plain text plus markdown-ish text in the main row. Enough for search and agent answers without extra blob storage.
D-003Use D1 FTS5 over app-side ranking. Native search is cheaper and simpler.
D-004Deploy as a Worker with static assets. Matches board; one deployment path.
D-005Make chat retrieval-first and deterministic. Ships without any extra model binding.

Deviations

IDFrom spec · Did · Reason
X-001Wanted agent-style answers; shipped retrieval-first cited matches; no model secret or binding required for deploy.

Tradeoffs

IDPicked over Rejected · Reason
T-001regex/readability-lite extraction over full parser dependency. Smaller diff.
T-002FTS only over embeddings on day 1. Good enough for MVP.
T-003default workers.dev deploy over custom domain setup. Zero extra infrastructure.

Assumptions

  • A-001 · Deploy to default workers.dev hostname. If wrong: add custom domain later.
