Algorithm

  1. Select link-backed notes with source_status missing/pending/failed, empty content, missing telemetry, or no chunk rows.
  2. Throttle pending rows until five minutes old and failed rows until twenty-four hours old.
  3. Build chunks from existing preview text when only chunk rows are missing.
  4. Refresh each selected weak source through the existing scraper and replace chunks.
  5. Run at most LINK_BACKFILL_LIMIT rows per cron tick.

Decisions

IDChoice · Rationale
D-001Reuse the hourly cron. No new scheduler surface.
D-002Default to 5 links per tick. Caps external fetch cost and Worker duration.
D-003Expose a manual endpoint. Lets the user fill missing rows immediately after migrations.

Tradeoffs

IDPicked over Rejected · Reason
T-001batch refresh over full corpus sweep. Avoids hammering external sites.
T-002retry failed daily over never retry. Handles transient blocks without hourly churn.

Reversibility

R-001 · medium. Revert the cron call, route, and service; already-backfilled note/chunk data remains compatible.
